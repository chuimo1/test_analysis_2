## 1. 어제까지 상황 정리

- 4단계 분석 → 2단계(Phase 1 추출 / Phase 2 종합)로 재구조화 완료 (`38b0671` 커밋)
- 4개 Gemini API 키 회전·폴오버 적용
- 그러나 배포 후 분석 호출 시:
  - 1차 시도: `429 Too Many Requests` 반복 (각 키마다 `5 RPM exceeded`)
  - 최종: `504 Gateway Timeout` (Vercel 60초 초과)
- 사용자가 새 키 4개를 4개의 다른 Google 계정에서 발급해 등록 → 그래도 동일 증상

## 2. 어제는 몰랐던 진짜 원인 (리뷰 §4-2)

```
const MODEL = 'models/gemini-3-flash-preview'
```

- 2026-04 현재 Google 공식 공개 모델은 **2.x 계열까지** (2.0-flash, 2.5-flash, 2.5-pro 등). **3.x는 미공개**.
- 또한 SDK 규약상 `getGenerativeModel({ model })`에 넘기는 값은 **접두사 `models/` 없이** 써야 함.
- 따라서 매 호출이 *모델 not found* → 폴오버 시도 누적 → 60초 초과.

**우리가 봤던 429는 모델 미존재 오류와 별도로 다른 키들이 진짜 RPM에 닿았을 수도 있고, 일부는 잘못된 호출 패턴 누적의 결과로 추정.**
