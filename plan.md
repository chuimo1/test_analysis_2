# 품격에듀 — AI 시험분석 서비스 설계도

> 작성일: 2026-04-13  
> 대상 독자: 비전공자도 읽을 수 있도록 쉽게 설명합니다.  
> 목적: 지금까지 만든 코드를 정리하고, 앞으로 어떻게 발전시킬지 방향을 잡기 위한 문서입니다.

---

## 1. 서비스 한 줄 요약

> "학원 강사가 시험지 사진을 찍어 올리면, AI가 자동으로 분석해주고, 그 결과를 실장(관리자)이 블로그에 올릴 수 있도록 복사해주는 서비스"

### 서비스 성장 단계

| 단계 | 대상 | 핵심 기능 |
|---|---|---|
| **단기** | 강사 + 실장 | 시험지 AI 분석 → 블로그 발행 |
| **중기** | + 원생·학부모 | 분석 결과 공유·열람 기능 |
| **장기** | 여러 학원 (SaaS) | 학원별 독립 계정, 월 구독 모델 |

> SaaS = 설치 없이 인터넷으로 구독해서 쓰는 서비스. A학원·B학원이 각자 가입해 독립적으로 데이터를 관리하는 구조.

## 2. 전체 사용자 흐름 (User Flow)

### 강사 계정 생성 — 두 가지 방식 모두 지원
- **방식 A (실장 직접 등록):** 실장이 관리자 화면에서 아이디·임시비밀번호 생성 → 강사에게 전달 → 강사가 첫 로그인 후 비밀번호 변경
- **방식 B (강사 직접 신청):** 강사가 가입 신청 → 실장이 승인

```
[강사]
  1. 계정 생성 (실장 직접 등록 OR 강사 신청 → 실장 승인)
        ↓
  2. 로그인 (아이디 + 담당과목 선택 + 비밀번호)
        ↓
  3. 시험지 업로드 (과목 자동 고정, 학년/학교/시험종류 입력)
        ↓
  4. AI가 분석 (Gemini API, 약 30~60초 소요)
        ↓
  5. 분석 결과 수정 (출제현황 표, 킬러문항, 전략 등 직접 수정 가능)
        ↓
  6. "수정 완료 · 실장에게 전달" 버튼 클릭

[실장(관리자)]
  7. 관리자 대시보드에서 완료된 분석 확인
        ↓
  8. 블로그 미리보기 페이지에서 HTML 복사
        ↓
  9. 외부 블로그(네이버 등)에 붙여넣기

[원생 / 학부모]
  10. 회원가입 후 로그인
        ↓
  11. 담당 강사·과목의 분석 결과 열람
```

### 확정 사항
- 실장 → 강사 수정 요청 기능: **불필요**
- 원생/학부모 공유: **회원가입 후 로그인하여 열람**
- 블로그 발행: **실장이 HTML 복사 → 네이버 블로그 등에 직접 붙여넣기**

## 3. 페이지 구조 (어떤 파일이 어떤 화면인지)

```
app/
├── page.tsx                        → "/" 접속 시 자동으로 /login으로 이동
├── login/page.tsx                  → 로그인 화면
├── signup/page.tsx                 → 강사 가입 신청 화면
│
├── teacher/                        ← 강사 전용 영역
│   ├── page.tsx                    → 강사 대시보드 (분석 목록)
│   ├── upload/page.tsx             → 시험지 업로드 + AI 분석 요청
│   └── analysis/[id]/page.tsx      → 분석 결과 확인 및 수정
│
├── admin/                          ← 관리자(실장) 전용 영역
│   ├── page.tsx                    → 관리자 대시보드 (분석 목록 + 가입 승인)
│   └── post/[id]/page.tsx          → 블로그용 미리보기 + 복사
│
└── api/                            ← 서버 코드 (백엔드)
    ├── analyze/route.ts            → Gemini AI 분석 요청 처리
    └── models/route.ts             → 사용 가능한 AI 모델 목록 조회
```

> 원생/학부모 페이지(`/student` 등)는 **나중에 추가** 예정. 현재 범위에서 제외.

## 4. 데이터 구조 (어떤 정보를 어떻게 저장하는가)

### 4-1. 현재 저장 방식 (임시)

> ⚠️ 현재는 **브라우저 localStorage** (컴퓨터 안에만 저장되는 임시 저장소)를 사용합니다.
> 실제 서비스에서는 Supabase(클라우드 데이터베이스)로 교체해야 합니다.

| localStorage 키 | 내용 | 저장 시점 |
|---|---|---|
| `currentUser` | 로그인한 사용자 정보 (아이디, 과목, 역할) | 로그인 시 |
| `analysisResult` | 가장 최근 AI 분석 결과 (임시) | 업로드 완료 시 |
| `examList` | 저장된 시험 분석 목록 전체 | 임시저장/완료 시 |
| `currentExamId` | 현재 편집 중인 시험 ID | 분석 저장 시 |
| `subjectChangeRequests` | 강사의 담당과목 변경 신청 목록 | 변경 신청 시 |

### 4-2. Supabase 테이블 설계 (예정)

| 테이블 | 주요 컬럼 |
|---|---|
| `users` | 아이디(고유값·중복불가), 이름, 전화번호, 담당과목, 승인여부, 역할(admin/teacher), 첫로그인여부, **활성여부(퇴사 강사 비활성화용)** |
| `exams` | 연도, 학교명, 학년, 과목, 시험종류, 시험범위(JSON), 예상난이도, 강사총평, 분석결과 전체, 강사ID, 완료여부, **발행여부** |
| `exam_files` | 시험ID, 파일URL, 파일종류(current/prev/solution), 업로드일시, 삭제예정일 |
| `subject_change_requests` | 강사ID, 현재과목, 신청과목, 신청일, 처리상태 |

### 확정 사항
- **역할:** `users` 테이블 하나에 역할 컬럼으로 실장/강사 구분 (admin/teacher)
- **삭제 권한:** 강사는 `수정 중` 상태일 때만 삭제 가능. 실장에게 전달(`수정 완료`) 후에는 삭제 불가
- **학원명:** 현재는 저장하지 않음 (SaaS 확장 시 추가 예정)
- **검색 필터:** 연도·학교명·학년·과목·시험종류로 충분

### 4-3. 핵심 데이터 모델 (`lib/types.ts`)

#### 문제(ExamQuestion) — 시험의 각 문항 정보
```
{
  number: 1,                    // 문제 번호
  type: '객관식',                // 문제 유형 (객관식/단답형/서술형)
  mainUnit: '문학',              // 대단원 (수학이면 "수와 연산" 등)
  subUnit: '현대시',             // 중단원
  source: '교과서',              // 출처 (교과서/부교재/모의고사/학습지/직접입력)
  intent: '주제찾기',            // 출제자 의도
  expectedCorrectRate: 82,      // 예상 정답률 (%)
  difficulty: '중',             // 난이도 (상/중상/중/중하/하)
  score: '-'                   // 배점 (없으면 "-")
}
```

#### 킬러문항(KillerQuestion) — 가장 어려운 문항들
```
{
  number: 8,           // 문제 번호
  subUnit: '비문학',    // 중단원
  intent: '함의찾기',   // 출제자 의도
  difficulty: '상',    // 난이도
  rate: 48,            // 예상 정답률 → 문제 테이블과 자동 동기화됨
  reason: '...'        // 킬러문항인 이유
}
```

#### 전략(StrategyItem) — 다음 시험 대비 전략
```
{
  trend: '함의찾기 비중 급증',   // 출제 경향
  strategy: '지문 핵심문 파악...' // 대응 전략
}
```

## 5. 각 화면 상세 설명

### 5-1. 로그인 (`/login`)

**무엇을 하는 화면인가?**
- **아이디 + 비밀번호만으로 로그인** (과목 선택 없음)
- 로그인 성공 시 DB에서 해당 사용자의 역할·담당과목 자동으로 불러옴
- 역할에 따라 강사 → `/teacher`, 실장 → `/admin`으로 이동

> 현재 코드에는 로그인 시 과목 선택 드롭다운이 있으나, Supabase 연동 시 제거 예정

**단점/한계 (현재):**
- ❌ 실제 인증 없음 — 아무 아이디/비밀번호나 입력해도 로그인됨 (Supabase 연동 전까지)
- ❌ 아이디에 "admin"이 들어가면 누구나 관리자 화면 접근 가능 → 보안 취약

---

### 5-2. 강사 계정 생성 (`/signup` + 관리자 직접 등록)

#### 방식 A — 강사 직접 신청 (`/signup`)
- 입력 항목: 실명, 전화번호, 담당과목, 아이디, 비밀번호(×2)
- 신청 후 실장 승인 전까지 로그인 불가
- 전화번호 자동 포맷팅, 비밀번호 8자 이상 검증

#### 방식 B — 실장 직접 등록 (관리자 화면)
- 실장이 강사 이름만 입력 → 아이디 자동 생성
- **초기 비밀번호: `3700` (고정)**
- 강사가 첫 로그인 시 필수 정보 입력 화면으로 강제 이동:
  - 전화번호, 담당과목, 새 비밀번호 설정 완료 후 서비스 이용 가능

**구현 예정:**
- ✅ 아이디 입력 시 실시간 중복 검사 (Supabase DB 조회)
- ✅ 가입 신청 내용 DB 저장
- ✅ 실장 승인 시 로그인 가능 상태로 변경

---

### 5-3. 강사 대시보드 (`/teacher`)

**무엇을 하는 화면인가?**
- 강사가 분석한 시험 목록을 볼 수 있는 메인 화면
- 각 시험을 클릭하면 분석 결과 페이지로 이동
- 상단에 담당과목 배지 표시 + "과목 변경 신청" 버튼

**상태 표시:**
- 🟡 `수정 중` — 강사가 임시저장한 상태
- 🟢 `수정 완료` — 강사가 실장에게 전달한 상태

**확정 사항:**
- 시험 제목 형식: `[국어] 2025년 ○○고 고2 1학기 중간고사`
- 삭제 버튼: `수정 중` 상태일 때만 표시 (수정 완료 후 삭제 불가)
- 필터: **학교별 · 학년별 · 연도별** (과목 필터는 불필요 — 강사 1인 1과목)

**단점/한계 (현재):**
- ❌ 다른 기기에서는 안 보임 (localStorage는 해당 브라우저에만 저장) → Supabase 연동으로 해결 예정

---

### 5-4. 시험지 업로드 (`/teacher/upload`)

**무엇을 하는 화면인가?**
- 시험 정보 입력 (과목 자동고정, 학년, 학교, 연도, 시험종류)
- 분석할 시험지 업로드 (1~10장 이상)
- 비교용 최근 3개년 시험지 업로드 (최대 30장, 선택)
- AI 분석 요청 → 완료되면 분석 결과 페이지로 이동

**업로드 화면 단계 순서 (확정):**

```
STEP 1. 시험 정보 입력
  - 학교명, 학년, 시험 연도, 시험 종류
  - 과목은 자동 고정 (로그인한 강사의 담당과목)

STEP 2. 시험 범위 입력
  - 행 추가 방식: [분류 선택 ▼] [상세 내용 입력]
  - 분류: 교과서 / 부교재 / 모의고사 / 학습지
  - 예: [교과서] [00교육 1장~3장]
       [부교재] [EBS 수능특강 1~3]
       [모의고사] [3모 1,5,7번]

STEP 3. 예상 난이도 + 강사 총평
  - 예상 난이도: 상 / 중상 / 중 / 중하 / 하 중 선택
  - 간단한 총평 메모 (AI 분석 시 참고용 자유 텍스트)

STEP 4. 시험지 업로드
  - 분석할 시험지: PDF 또는 JPG/JPEG, 1~10장 이상
  - 비교용 전년도 시험지: PDF 또는 JPG/JPEG, 최대 30장 (선택)

STEP 5. AI 분석 버튼
  → 위 모든 정보를 Gemini에 전달하여 분석
```

**확정 사항:**
- 지원 파일 형식: **PDF + JPG/JPEG** 만 허용
- 과목 고정: 로그인한 강사의 담당과목 자동 적용 (변경 불가)
- 백그라운드 분석: 다른 페이지로 이동해도 분석 계속 진행 → Supabase 연동 후 구현
- 개별 문항 재분석: 분석 결과 화면에서 특정 문항만 AI 재분석 가능

**단점/한계 (현재):**
- ❌ 분석 소요시간 30~60초 (3단계 분리 요청)
- ❌ 브라우저 닫으면 분석 결과 사라짐 → Supabase 연동으로 해결 예정

---

### 5-5. 분석 결과 수정 (`/teacher/analysis/[id]`)

**무엇을 하는 화면인가?**
- AI가 분석한 결과를 강사가 직접 수정할 수 있는 화면
- 크게 5개 섹션으로 구성:

#### ① 출제 현황 테이블
- 모든 문항을 표로 표시
- 각 셀을 직접 클릭해서 수정 가능
- **문항 추가/삭제 가능** (AI 누락·오인식 시 강사가 직접 수정)
- 과목에 따라 컬럼이 다름 (아래 토의 내용 참고)

| 과목 | 컬럼 순서 |
|---|---|
| **수학** | 문제번호 \| 유형 \| 출처 \| 대단원 \| 중단원 \| 출제자의도 \| 예상정답률 \| 난이도 \| 배점 |
| **영어** | 문제번호 \| 유형 \| 출처 \| 영역(문법/독해/어휘/듣기) \| 출제자의도 \| 예상정답률 \| 난이도 \| 배점 |
| **국어** | 문제번호 \| 유형 \| 출처 \| 영역(문학/독서/문법/화법과작문/언어와매체) \| 작품명 \| 출제자의도 \| 예상정답률 \| 난이도 \| 배점 |
| **사회** | 문제번호 \| 유형 \| 출처 \| 대단원 \| 중단원 \| 출제자의도 \| 예상정답률 \| 난이도 \| 배점 |

**출처 입력 방식 (공통):**
- 대분류 드롭다운: `교과서 / 부교재 / 모의고사 / 학습지`
- 세부 텍스트 입력: 예) `EBS 수능특강`, `26년도 3월 모의고사`, `미래엔 3단원`

#### ② 차트 (과목별 구성)

> 차트는 과목 특성에 맞게 개수와 종류가 다름. 모든 차트는 실시간으로 테이블 수정 내용을 반영함.

| 과목 | 차트 구성 |
|---|---|
| **수학** | ①난이도(원형) ②출처(막대) ③대단원별 누적막대(중단원 쌓음) |
| **영어** | ①난이도 ②출처 ③영역(문법/독해/어휘/듣기) ④세부유형 |
| **국어** | ①난이도 ②출처 ③영역(문학/독서/문법/화작/언매) ④세부유형 |
| **사회** | ①난이도 ②출처 ③대단원별 누적막대 ④세부유형 |

- 📊 **난이도별 비중** (공통) — 원형 차트, 상/중상/중/중하/하 비율
- 📂 **출처별 분석** (공통) — 막대 차트, 교과서/부교재/모의고사/학습지/직접입력
- 📚 **대단원별 누적막대** (수학·사회) — 대단원 안에 중단원을 쌓아서 표시 (중단원이 많아도 가독성 유지)
- 🗂️ **영역별 분석** (영어·국어) — 과목 특성상 대단원 개념이 없어 '영역' 기준
- 🏷️ **세부유형별 분석** (영어·국어·사회) — AI가 분류한 출제의도 세부 유형

#### ③ 시험 총평
- 주요 특징 4개 (텍스트 직접 수정)
- 전년도 비교 (여러 줄 텍스트 직접 수정)

#### ④ 킬러문항
- **AI 자동 선정**: 난이도 "상" 문항 중 정답률 낮은 순
- **강사 수동 추가/교체 가능**: AI 원안에서 특정 문항 제거하거나 다른 문항을 킬러로 지정
- **최대 5개까지 가변**: 시험 난이도에 따라 강사가 1~5개 범위에서 조절
- **정답률은 ①번 테이블과 자동 동기화** (따로 입력 불필요)
- 단원, 출제의도, 난이도, 이유 수정 가능
- **손풀이 이미지 첨부 가능** (강사가 직접 푼 풀이 사진 업로드)

#### ⑤ 다음 시험 전략
- 출제경향 + 대응전략 쌍으로 구성 (AI 원안 4~5개)
- **강사가 항목 추가/삭제/순서변경 자유** — AI가 준 항목을 기반으로 강사 판단에 따라 편집

**저장 방식:**
- `임시 저장` → localStorage에 `isFinalized: false`로 저장
- `수정 완료·실장에게 전달` → localStorage에 `isFinalized: true`로 저장 + 알림

**블로그 미리보기:**
- 강사도 분석 결과 화면에서 바로 블로그 미리보기 확인 및 수정 가능
- 실장은 최종 점검 후 HTML 복사 → 블로그 발행

**장점:**
- AI가 틀린 내용을 강사가 직접 고칠 수 있어 정확도를 높일 수 있음
- 킬러문항 정답률이 테이블과 자동으로 맞춰짐 → 불일치 방지
- 차트가 실시간 반영되어 수정 결과를 바로 시각적으로 확인 가능

**단점/한계:**
- ❌ 문항 추가/삭제 기능 없음 (AI가 누락한 문항을 수동으로 추가 불가)
- ❌ 블로그 HTML 자동 생성 기능 미구현 (어드민 페이지의 블로그 내용이 아직 더미 데이터)
- ❌ 킬러문항 `+ 손풀이` 버튼이 있으나 기능 미구현

### 5-6. 관리자 대시보드 (`/admin`)

**무엇을 하는 화면인가?**
- 탭 구조로 **4가지 기능**:

#### 탭 1: 시험 분석 목록
- 강사들이 제출한 분석 결과 목록
- **상단 상태탭**으로 우선 분류:
  - 🔴 **발행 대기** — 강사 수정완료 + 블로그 미발행 (실장 주업무 대상)
  - 🟡 **수정 중** — 강사 작업중 (기다려야 할 것)
  - ✅ **발행 완료** — 블로그까지 올라감 (기록 보관)
- **보조 필터**: 학교 · 학년 · 과목 · 연도 · 시험종류
- **정렬**: 최신순 (기본)

#### 탭 2: 가입 승인
- 강사가 신청한 가입 목록
- 승인/거절 버튼 (승인하면 목록에서 사라짐)

#### 탭 3: 과목 변경 승인 (신규)
- 강사가 담당과목 변경을 신청한 목록
- 현재 과목 → 신청 과목 표시
- 승인/거절 버튼

#### 탭 4: 강사 관리 (신규)
- 등록된 강사 전체 목록 + 상태(활성/비활성)
- **비활성화 / 재활성화 버튼**:
  - 퇴사 강사 등의 로그인을 차단하되 기존 분석 데이터는 보존
  - 언제든 재활성화 가능
- 강사 기본 정보(이름, 전화번호, 담당과목, 가입일) 조회

**장점:**
- 관리자가 강사 가입·변경·활성화 상태를 직접 컨트롤
- 상태탭 구조로 "발행 누락" 실수 최소화 (시험 당일 업로드 워크플로우 지원)

**단점/한계:**
- ❌ 현재 시험 목록이 더미 데이터 — localStorage에서 읽지 않음
- ❌ 가입 승인해도 실제로 로그인 가능해지지 않음 (DB 없기 때문)
- ❌ 과목 변경 신청 탭 미구현 (localStorage에 저장은 되지만 관리자가 볼 수 없음)
- ❌ 강사 관리 탭 미구현

### 5-7. 블로그 미리보기 (`/admin/post/[id]` + `/teacher/analysis/[id]`)

**무엇을 하는 화면인가?**
- 분석 결과를 블로그 형식으로 미리 보는 화면
- **강사도 자신의 분석 화면에서 블로그 미리보기 확인·수정 가능** (실장은 최종 점검자)
- 텍스트 복사 / HTML 복사 버튼 제공

**대상 플랫폼: 네이버 블로그 전용**
- 네이버 에디터가 지원하는 HTML 범위로 제한해서 생성
- 티스토리 등 다른 플랫폼은 지원 범위 외

**에디터 동작 방식:**
- **자동 생성**: 분석 결과(JSON) → 블로그 HTML로 자동 변환
- **전체 자유편집 (리치텍스트 에디터)**: 생성된 본문 전체를 자유롭게 편집 가능
  - 머리말/인사말/본문/표/차트/마무리 모두 편집
  - 데이터와 본문이 불일치할 가능성 있음 (편집한 내용은 별도 저장)

**장점:**
- 네이버 블로그에 바로 붙여넣기 가능한 HTML 형태 제공
- 강사가 미리 확인·수정 가능 → 실장 업무 부담 감소

**단점/한계:**
- ❌ 분석 결과를 실제로 불러오지 않음 — 아직 더미(가짜) HTML이 고정으로 표시됨
- ❌ 블로그 내용 생성 로직 없음 (분석 데이터 → HTML 변환 기능 미구현)
- ❌ 리치텍스트 에디터 미도입

### 5-8. AI 분석 API (`/api/analyze`)

**무엇을 하는 코드인가?**
- 강사가 업로드한 시험지 이미지를 Google Gemini AI에게 보내서 분석을 요청하는 서버 코드
- 한 번에 모든 걸 요청하면 오류(용량 초과)가 나서 **4단계로 나눠서 요청**:

```
1단계: 문제별 분석 (출제현황 테이블 데이터)
2단계: 시험 총평
3단계: 킬러문항
4단계: 다음 시험 전략
```

**에러 처리 정책:**
- **부분 성공도 저장**: 1~3단계만 성공하고 4단계 실패해도 성공한 만큼 저장
- **실패 부분만 재시도**: 강사가 "이 부분만 다시 분석" 버튼으로 실패 단계만 재요청
- 비용 절감 + 사용자 대기 시간 최소화

**과목별 AI 프롬프트 차이:**
| 과목 | 대단원 예시 | 출제자 의도 예시 |
|---|---|---|
| 국어 | 문학/독서/문법 | 주제찾기, 내용파악, 함의찾기 |
| 영어 | 문법/독해/어휘/듣기 | 빈칸채우기, 주제파악, 문법이해 |
| 수학 | 수와 연산, 함수, 기하 | 개념이해, 계산하기, 공식적용 |
| 사회 | 지리, 역사, 일반사회 | 개념파악, 연도파악, 원인분석 |

**현재 사용 AI 모델:** `models/gemini-3-flash-preview`

**장점:**
- 과목별로 다른 프롬프트를 써서 분석 정확도를 높임
- 4단계 분리로 응답 안정성 향상 + 단계별 독립 재시도 가능

**단점/한계:**
- ❌ 이미지 1장만 처리 (여러 페이지 시험지 불가) — 1~10장 지원 필요
- ❌ 부분 재시도 로직 미구현
- ❌ API 비용 발생 (호출할 때마다 Gemini API 요금)

| 자료 | 보관 정책 |
|---|---|

## 7. 현재 코드의 전체 장단점 총평

### ✅ 잘 된 점

1. **과목별 맞춤 UI** — 국어/영어는 출처 위주, 수학/사회는 단원 위주로 표가 달라짐
2. **킬러문항 정답률 자동 동기화** — 테이블 수정 시 킬러문항에도 자동 반영
3. **차트 실시간 반영** — 표를 수정하면 4개 차트가 바로 바뀜
4. **로그인 담당과목 고정** — 강사 실수로 다른 과목 분석 방지
5. **3단계 AI 요청 분리** — 한 번에 요청 시 발생하는 오류/타임아웃 방지
6. **임시저장 + 최종전달 분리** — 강사가 실수로 미완성본 전달하는 것 방지

### ❌ 개선이 필요한 점

1. **데이터가 브라우저에만 존재** — localStorage는 브라우저를 지우면 사라짐. Supabase 연동 필수
2. **실제 인증 없음** — 아무나 강사/관리자 페이지 접근 가능. 로그인 보안 구현 필요
3. **어드민 페이지가 더미 데이터** — 관리자가 실제 분석 결과를 볼 수 없음
4. **블로그 HTML 자동 생성 없음** — 분석 데이터 → 블로그 글 변환 기능 미완성
5. **이미지 1장 제한** — 여러 페이지 시험지 처리 불가
6. **과목 변경 신청이 관리자에게 안 보임** — localStorage 저장만 되고 관리자 탭 미구현

## 8. 미구현 기능 — 의논이 필요한 항목들

> 아래 기능들은 구현 방향에 대해 결정이 필요합니다.

---

### 🔴 우선순위 높음 (서비스에 필수적인 것들)

#### A. Supabase 데이터베이스 연동
- **현황:** localStorage에 임시 저장 중
- **문제:** 다른 기기에서 보이지 않음, 브라우저 초기화 시 데이터 손실
- **필요한 것:**
  - `users` 테이블 (강사 정보, 승인 여부)
  - `exams` 테이블 (시험 분석 결과 전체)
  - `subject_change_requests` 테이블 (과목 변경 신청)
- **의논할 점:** Supabase 무료 플랜으로 시작할 것인지?

#### B. 실제 로그인 인증 (Supabase Auth)
- **현황:** 아이디/비밀번호 입력해도 실제 검증 없음
- **문제:** 보안 없음 — 누구나 접근 가능
- **의논할 점:** 소셜 로그인(카카오/구글)도 추가할지?

#### C. 관리자 페이지에서 실제 데이터 연동
- **현황:** 더미 데이터만 표시
- **필요한 것:** localStorage 또는 Supabase에서 실제 분석 목록 읽기

---

### 🟡 우선순위 중간 (있으면 좋은 것들)

#### D. 블로그 HTML 자동 생성 + 리치텍스트 에디터
- **현황:** 블로그 미리보기 페이지에 더미 HTML이 고정 표시
- **결정:** 네이버 블로그 전용 + 전체 자유편집(리치텍스트)
- **필요한 것:**
  - 분석 결과 데이터(JSON) → 네이버 호환 HTML로 자동 변환 함수
  - TipTap 등 리치텍스트 에디터 도입

#### E. 과목 변경 신청 + 강사 관리 탭
- **현황:** 관리자 대시보드는 2탭만 있음
- **결정:** 4탭 구조 (시험분석 / 가입승인 / 과목변경 / 강사관리)
- **강사 관리:** 활성/비활성 토글로 퇴사 강사 로그인 차단(데이터는 보존)

#### F. 문항 추가/삭제 + 개별 재분석
- **현황:** AI가 분석한 문항만 표시. 추가나 삭제 불가
- **결정:** 행 추가/삭제 버튼 + "이 문항만 AI 재분석" 버튼 도입
- **문제:** AI가 문항 하나를 빠뜨리거나 잘못 인식하면 강사가 수정 불가

#### G. 여러 페이지 시험지 지원 (PDF + JPG/JPEG, 1~10장)
- **현황:** 이미지 1장만 분석 가능
- **결정:** 분석용 1~10장, 비교용 전년도 최대 30장 지원
- **지원 형식:** PDF + JPG/JPEG

#### H. AI 4단계 분리 + 부분 재시도
- **현황:** 3단계 분리, 전체 실패 시 전체 재요청
- **결정:** 4단계(문제/총평/킬러/전략) 분리 + 실패 단계만 재시도

#### I. 시험범위·예상난이도 입력 단계
- **현황:** 업로드 시 시험 정보만 입력
- **결정:** 업로드 화면에 STEP 2(시험범위 분류별 입력) + STEP 3(예상난이도+총평) 추가

#### J. 파일 자동삭제 스케줄러
- **현황:** 모든 파일 영구 보존 가정
- **결정:** 발행 시 분석용 파일 삭제 / 임시저장 12시간 / 손풀이 3일
- **구현:** Supabase Edge Function 또는 cron
